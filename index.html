Geo. Predicates Working Group

The Samvera Geospatial Predicates Working Group was chartered during June 2017 in order to determine a set of best practices and recommendations regarding the encoding of geospatial repository metadata as linked data. Samvera repositories aim to publish content descriptive metadata as linked data using the Resource Description Framework (RDF), and those participating in this working group sought out to determine precisely which RDF ontologies would be best suited for linking descriptions for geospatial repository resources.

This section describes the status of this document at the time of its publication. Other documents may supersede this document. A list of current releases and the latest revision of these recommendations can be found on the Samvera Community Wiki.

This document was published by the Samvera Geo. Predicates Working Group as a community recommendation for RDF usage in Samvera repositories.

All interested parties are invited to provide implementation and bug reports and other comments through the Working Group's GitHub repository. These will generally be considered in the development of future releases for this document.

Namespaces and Document Conventions

The names of RDF entities -- classes, predicates, individuals -- are URIs. These are usually expressed using a compact notation where the name is written prefix:localname, and where the prefix identifies a namespace URI. The namespace identified by the prefix is prepended to the localname to obtain the full URI.

The following namespaces are used in this document:

dcterms http://purl.org/dc/terms/

All RDF examples are written in Turtle syntax [turtle].

Target Attributes

Scope Name Definition Representation
Geographic Extent The computer-actionable description of a geographic location (point, line or area) on the surface of the Earth. This can be used to describe the extent on the Earth represented by an object (such as the extent of a shapefile). This can also be used to describe the extent on the Earth of a geographic location as a subject term (see geographic location definition below). The geographic extent can be represented as an area (bounding box or footprint), line (path) or point
Geographic Extent Bounding box (rectangle) The geographic extent expressed in terms of the greatest extent of latitude and longitude (the farthest west, east, north, and south an object extends) Usually expressed in decimal degrees (computer actionable) and/or degrees minutes seconds (DMS) (human readable)
Geographic Extent Polygon The geographic extent representing the footprint of the object expressed as a multi-sided polygon (area) of the actual area represented by the data. The polygon may include “donut holes” within the polygon and/or may include more than one polygon. The polygon may be a bounding box that is skewed and/or not represented using latitude and longitude. Usually recorded as a series of x, y coordinate pairs using decimal degrees (computer actionable). May also be expressed as decimal numbers where the units are meters or feet if the coordinate pairs are projected.
Geographic Extent Path The geographic extent expressed as a line representing the path along which data were collected or objects were created series of (2:n) coordinate pairs (nodes), usually expressed as latitude and longitude in decimal degrees from a start point to an end point.
Geographic Extent Point The latitude and longitude coordinate pair representing a geographic extent as a point on the surface of the Earth. The coordinate pair may represent the exact or approximate physical location or may represent the centroid of an area in which the object resides (such as a city, county, etc.) Usually recorded as two separate elements; latitude and longitude, each usually expressed in decimal degrees (computer actionable) and/or degrees minutes seconds (DMS) (human readable)
Geographic Location The human-readable place name representing an exact or approximate physical location on the surface of the Earth. This can be used to describe the location on the Earth representing the extent of an object, a subject, or any other use of a place name. The geographic location can be represented by a location place name or free-text description
Geographic Location Place name The name of a geographical location, such as a city, mountain, lake, etc. text string. The name may or may not be based on a controlled vocabulary. The name may have variations based on language, time period, source of information, etc.
Geographic Location Description The description of a geographic location using way points or features. text string (free text). Example: “corner of 4th Street and Mason Ave.”.
Scale The scale of an object is the relationship between a measured distance on the object and the corresponding actual distance on the surface of the Earth.
Scale Representative fraction denominator The value to the right of the colon in a ratio of equal units representing the scale (i.e. 1:400,000 means that one inch on the object equals 400,000 inches on the ground, as well as one centimeter on the object equals 400,000 centimeters on the ground) Integer that is the denominator of a representative fraction (‘300000’ to represent 1:300,000)
Scale Text A written statement of the scale A text string usually recording what is written on the object (1 inch = 100 miles, not to scale, etc.) The text string may be the representative fraction in its entirety (‘1:300,000’) as opposed to just the denominator.
Scale Qualifier Used to clarify the accuracy or reliability of the representative fraction or text string (approximate, inaccurate, etc.) Text string, could be from a controlled vocabulary
Spatial Reference System The spatial reference system is the mathematical construct of a geometric system used to locate objects on a three-dimensional surface (such as the Earth). Could be an EPSG code or could be a formal name of a projection. Possibly expressed as a combination of projection, coordinate system, and datum.
Spatial Data Representation The graphical characteristics of the data in terms of display.
Spatial Data Representation Spatial Data Representation Type The general method used to organize and display the graphic display of the data, either through vertices and paths (vector), through grid cells (raster), or through text (geoJSON). Vector, Raster, geoJSON
Spatial Data Representation Spatial Data Feature Type The type of vertices and paths used to display the data, either vertices alone (point), paths connecting a series of vertices (line), or a closed path connecting a series of vertices where the first and last point are the same and the object has an inside and an outside (polygon). point, line, polygon
File Format for Geospatial Objects The encoding mechanism used to store the geospatial data A valid file format expressed as a term (as opposed to a file extension), such as shapefile, GeoTIFF, etc.
Flight Altitude Used with remotely sensed images; the height above the ground from which the sensor (camera) is located when the data was collected (i.e. when the photo was taken). This element does not represent a geographic location’s elevation above sea level. integer (usually in feet or meters), possibly with an accompanying element to indicate the units (e.g. ‘feet’, ‘meters’), or expressed as a text string with the units or a symbol of units included (e.g. 7200’)
Raster Resolution The size of a grid cell in a raster dataset as a measure of the actual distance on the surface of the Earth that is represented by one side of the cell. This is different from the pixel resolution associated with digital images. Text string indicating a distance (e.g. 30 meters)
Remote Sensing Details Name or description of a remote sensing platform used to acquire a set of geospatial data. Text string

Example Usage

Scanned Map

# Define the raster data set as an instance of schema:Map

testRepository:exampleScannedMap a pcdm:Work, schema:Map ;
  dcterms:spatial    myGazetteer:place1 ;
  dcterms:description    "An Example Scanned Map"@en ;
  schema:spatialCoverage    myGazetteer:place1 ;
  ebucore:hasMimeType    "image/tiff" .
